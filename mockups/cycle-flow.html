ROOT CAUSE
The ╌ (box drawings light double dash horizontal, U+254C) does NOT connect flush in most monospace fonts. Each glyph has visible whitespace on both ends. Same problem with ┄ ┈ and other dashed box-drawing chars. This is a font issue, not a code issue — no fix possible with these characters.

RENDERING RULES
1. No dashed box-drawing chars — ╌ ┄ ┈ all have gaps. Use dim solid ─ or no box.
2. Zero blank lines — connectors flow directly from box bottom to next element.
3. Use ┬ ┴ for connections — branch lines connect via T-junctions directly into box top/bottom edges. No free-floating │ lines between elements.
4. Box width = fixed per row — all agents in a row are same width (widest name + padding). Prevents column misalignment.
IMPLEMENTATION: CONNECTOR MATH
Each agent box is boxW chars wide. Total row width = N × boxW. Center column = floor(totalWidth / 2). Branch T-junction positions = center of each box = i × boxW + floor(boxW / 2). The ┬/┴ chars in the box top/bottom edges are placed at those exact columns. No separate connector lines needed — the box borders ARE the connectors.

DIMMING LEVELS
Two tiers of dimming: Cycles older than N-1 render at opacity ~0.35 (deeply faded). The immediately previous cycle (N-1) renders at ~0.5 (dimmed but legible). Current cycle renders at full brightness with colored borders and bold status icons. [F] label changes to "collapse" when expanded.
SCROLL BEHAVIOR
When expanded, the flow auto-scrolls to keep the current cycle visible at the bottom of the panel. User can scroll up to see older cycles. Scroll position resets on cycle change.
