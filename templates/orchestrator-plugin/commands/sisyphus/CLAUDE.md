# templates/orchestrator-plugin/commands/sisyphus/

## `spec` — Agent Replacement

`sisyphus:spec` fully replaces `sisyphus:requirements` and `sisyphus:design` — those agents no longer exist and will error if spawned. If the current `strategy.md` lists separate requirements/design stages, collapse them into a single spec stage **before** spawning.

Outputs land at `context/design.md`, `context/design.json`, `context/requirements.json`, `context/requirements.md`. `requirements.md` is generated by a pure-code script inside the agent — no LLM call for that step.

## `strategize` — Session Must Be Active

If `sisyphus status` shows the session completed, run `sisyphus continue` before yielding. Yielding from a completed session is a no-op.
