# Skill Format Reference

Target: /Users/silasrhyneer/Code/crouton-kit/plugins/authoring/skills/{skill-name}/

## SKILL.md (~150-200 lines)
- YAML frontmatter: `name`, `description` (include trigger keywords for discovery)
- Overview section explaining the core concept
- Practical sections with principles, patterns, common mistakes
- Links to reference.md for depth
- Match tone/structure of output-variety/SKILL.md and prompting-effectively/SKILL.md

## reference.md (~150-250 lines)
- Implementation patterns with code examples (TypeScript/Python)
- Research citations inline: papers, blog posts, Twitter/X, Reddit/HN
- Concrete examples showing good vs bad patterns
- Match tone/structure of output-variety/reference.md

## Style
- Practitioner voice — direct, opinionated, grounded in evidence
- No fluff or hedging. State what works and what doesn't.
- Code examples should be realistic, not toy
- Citations: "[Author (Year) — Title](URL)" or "Author's Blog: [Title](URL)"
- Every major claim should have a source
